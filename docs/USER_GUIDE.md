# Super Markdown Workbench 使用手册

## 1. 项目简介

Super Markdown Workbench 是一个本地优先的在线 Markdown 工作台，适合需要频繁处理本地 `.md` 文件的开发者和内容工作者。它提供轻量首页入口、实时预览、自动保存、恢复草稿、导出、主题切换和中英双语界面。

## 2. 适用场景

- 快速打开并编辑本地 Markdown 文件
- 在浏览器里持续写作并实时预览
- 页面刷新或误关后恢复最近编辑内容
- 导出为 `.md` 或 `.html`
- 在中文和英文界面之间切换
- 让 Editor 或 Preview 单独全屏工作

## 3. 启动项目

在项目根目录执行：

```bash
npm install
npm run dev
```

默认访问地址：`http://localhost:3000`

生产构建：

```bash
npm run build
npm start
```

## 4. 首页使用说明

首页提供 5 类主要入口与操作：

- `新建文档`：创建一个空白 Markdown 文档并进入编辑器
- `导入 Markdown`：选择本地 `.md` 文件，导入后直接进入编辑器
- `继续草稿`：当存在可恢复草稿时，继续上次未完成编辑
- `最近文档`：展示最近打开过的本地文档，点击 `open` 进入编辑器
- `English / 中文`：切换首页和编辑器的界面语言

## 5. 编辑器使用说明

编辑器页面包含 2 个核心区域：

- `Editor`：左侧 Markdown 编辑区，基于 CodeMirror
- `Preview`：右侧实时渲染预览区

顶部工具栏支持：

- `中文 / English`：切换界面语言
- `Dark Mode / Light Mode`：切换主题
- `Editor Fullscreen / Preview Fullscreen`：让编辑区或预览区一键全屏
- `Exit Fullscreen`：退出全屏并恢复双栏布局
- `Export HTML`：导出当前文档为 HTML
- `Export Markdown`：导出当前文档为 Markdown
- `Saved / Saving / Dirty / Error`：显示当前保存状态

## 5.1 双栏宽度调整

- 编辑区和预览区之间支持鼠标拖拽调节宽度
- 拖拽后的宽度会保存到浏览器本地
- 刷新页面后会尽量恢复上次设置
- 进入任一全屏模式后，会临时只保留当前主区域；退出后回到双栏布局

## 6. 自动保存与恢复机制

- 编辑内容后会自动保存到浏览器本地存储
- 默认采用防抖保存，避免频繁写入
- 如果浏览器被刷新或意外关闭，下次进入首页会显示恢复入口
- 恢复草稿后会回到对应编辑器页面

## 7. 导入与导出

### 导入

- 仅支持从首页导入本地 `.md`
- 文件名会作为初始文档标题
- 导入后自动创建本地文档记录

### 导出

- `Export Markdown`：导出原始 Markdown 内容
- `Export HTML`：导出带样式的完整 HTML 文档

## 8. 本地数据存储说明

项目采用本地优先存储策略：

- `IndexedDB`：保存文档正文、草稿数据
- `localStorage`：保存主题、语言和布局等轻量偏好

这意味着：

- 不需要登录
- 数据默认只保存在当前浏览器环境
- 清空浏览器站点数据后，本地文档会丢失

## 10. 当前版本限制

当前版本暂不支持：

- 账号体系
- 云同步
- 多人协作
- 全文搜索
- 标签 / 文件夹
- AI 自动改写或总结

## 11. 常见问题

### Q1：为什么我换浏览器后看不到之前的文档？

因为当前版本是本地优先，数据只保存在当前浏览器本地。

### Q2：为什么我清空浏览器缓存后文档没了？

因为文档保存在浏览器本地存储中，清除站点数据会一并删除。

### Q3：导入文件后为什么会创建新文档？

这是当前版本的既定行为，导入始终生成一个新的本地文档，不会覆盖已有文档。
